# 管理费结算标准文档

## 更新日期
2026-03-16

## 重要说明

**管理费与佣金的区别**：
- **管理费**：仅与服务用户的个数和服务日期有关，包括健康教练管理费、师父监管费、医生管理费
- **佣金**：与销售成交有关，包括线下成交提成、线上成交提成，由佣金结算模块独立处理

本文档仅说明管理费结算标准。销转提成规则请参考《佣金结算标准文档》。

## 1. 健康教练管理费结算标准

### 1.1 前三个月管理费
- **结算标准**: 每月400元
- **适用对象**: 管理未满3个月的新用户
- **结算周期**: 从管理开始计算的前三个月，每月10日结算
- **计算公式**: `管理天数 × (400元 ÷ 30天)`

### 1.2 三个月后管理费
- **结算标准**: 每月100元
- **适用对象**: 用户管理满三个月的健康教练
- **结算周期**: 按月结算，每月10日结算
- **计算公式**: `管理天数 × (100元 ÷ 30天)`

### 1.3 师父监管费
- **结算标准**: 每月20%的提成
- **前三个月**: 80元/月 (400元的20%)
- **后九个月**: 20元/月 (100元的20%)
- **全年总计**: 420元
- **适用对象**: 师父角色的健康教练
- **结算周期**: 按月结算，每月10日结算
- **计算方式**:
  - 前三个月：`管理天数 × (80元 ÷ 30天)`
  - 后九个月：`管理天数 × (20元 ÷ 30天)`

### 计算示例
```
假设某师傅管理10个用户：
- 新用户5人（管理时间都在前3个月内）
- 老用户5人（管理时间都超过3个月）

当月管理费计算：
1. 基础管理费：
   - 新用户：5人 × 400元 = 2000元
   - 老用户：5人 × 100元 = 500元
   - 基础管理费总计：2500元

2. 师父监管费：
   - 新用户：5人 × 80元 = 400元
   - 老用户：5人 × 20元 = 100元
   - 师父监管费总计：500元

3. 当月总管理费：2500 + 500 = 3000元
```

## 2. 医生管理费结算标准

### 2.1 医生管理费
- **结算标准**: 一年3000元
- **费用包含**:
  1. 一年周期的群内守护回复消息
  2. 前三个月，每28天一次复盘电话
  3. 后九个月，每一个季度结束一次复盘电话
- **适用条件**: 用户年度管理周期，有医生管理的情况
- **结算周期**: 用户付费满一个月后无退费情况，一次性结清全年管理费

### 结算条件
- 用户首次付费日期已满一个月
- 用户未发生退费
- 每个用户只能结算一次医生管理费

### 计算示例
```
假设某医生管理3个用户，都符合结算条件：

医生管理费收入：
- 3人 × 3000元 = 9000元（一次性结算）
```

## 3. 销转提成结算（参考信息）

**注意**: 以下销转提成规则仅供参考。实际提成计算由佣金结算模块独立处理，不属于管理费系统。

### 3.1 线下成交提成10%
- **结算标准**: 医生5%，销售5%，成交客户市场人员服务费200元
- **分配方案**:
  - 医生提成：`成交金额 × 5%`
  - 销售提成：`成交金额 × 5%`
  - 市场人员服务费：200元（固定）
- **总计**: 成交金额的10% + 200元服务费

### 3.2 线上成交提成10%
- **结算标准**: 销售5%，医生3%，销售2%
- **分配方案**:
  - 销售主要提成：`成交金额 × 5%`
  - 医生提成：`成交金额 × 3%`
  - 销售辅助提成：`成交金额 × 2%`
- **总计**: 成交金额的10%

### 计算示例
```
线下成交5000元：
- 医生提成：5000 × 5% = 250元
- 销售提成：5000 × 5% = 250元
- 市场人员服务费：200元
- 总支付：250 + 250 + 200 = 700元

线上成交5000元：
- 销售主要提成：5000 × 5% = 250元
- 医生提成：5000 × 3% = 150元
- 销售辅助提成：5000 × 2% = 100元
- 总支付：250 + 150 + 100 = 500元
```

## 4. 结算注意事项

### 4.1 提报时间
- 请各部健康教练核对对管理用户信息
- 每月7日前提报上个月结算用户

### 4.2 医生上线次数统计
- 医生三个月后的上线次数需健康教练统计
- 以便准确记录上线次数

### 4.3 管理时间计算
- 教练管理时间从首次管理日期开始计算
- 按实际管理天数计算费用

### 4.4 特殊情况处理
- 如有特殊情况，请及时与耿爽沟通

## 5. 技术实现

### 5.1 数据结构
管理费记录包含以下信息：
- 费用类型（新用户/老用户/师父监管/医生）
- 收款人信息
- 管理用户信息
- 费用金额
- 结算周期
- 状态（待确认/已确认/已支付）

**重要说明**: 销转提成属于佣金结算模块，与管理费分离。管理费仅与用户服务个数和服务日期有关。

### 5.2 计算函数
```typescript
// 健康教练管理费
addManagementFee(
  employeeId, employeeName,
  clientId, clientName,
  isNewClient,
  managementStartDate,
  period,
  mentorId?  // 师父ID（用于师父监管费）
)

// 医生管理费
addDoctorManagementFee(
  doctorId, doctorName,
  clientId, clientName,
  firstPaymentDate
)
```

### 5.3 自动计算规则
1. 系统自动计算管理天数
2. 根据管理天数判断新/老用户
3. 自动识别师傅身份并计算师父监管费
4. **注意**: 销转提成不在管理费系统内计算，由佣金结算模块独立处理
