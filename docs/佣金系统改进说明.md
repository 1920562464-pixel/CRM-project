# 佣金系统改进说明

## 更新日期
2026-03-16

## 改进内容

### 1. ✅ 支持教练作为销售人员

**需求背景**：教练不仅可以服务用户获得管理费，还可以参与销售获得佣金提成。

**实现方式**：
- 在销转提成功能中，`salesId` 参数可以接受教练ID
- 系统自动识别员工角色，如果是教练则使用 `'coach'` 作为 `employeeRole`
- 教练获得的销售提成比例与销售人员相同

**代码示例**：
```typescript
// 教练张三参与销售
const records = addSalesCommissionByProduct(
  'TXN001',
  'C001',
  '客户姓名',
  'offline',
  'L2标准管理',
  '2026-03-15',
  'DOC002',      // 医生ID
  'COACH001'     // 教练ID（作为销售人员）
)

// 生成的佣金记录中：
// - employeeRole: 'coach' （而不是 'sales'）
// - 仍然获得5%的销售提成
```

### 2. ✅ 产品价格配置（规范化佣金添加）

**需求背景**：为了减少手动输入金额的错误，需要预置标准产品价格。

**实现方式**：
- 创建 `PRODUCT_PRICE_CONFIG` 配置对象
- 包含线下和线上两类产品价格
- 新增 `addSalesCommissionByProduct()` 函数，通过产品类型自动获取价格
- 保留 `addSalesCommission()` 函数，支持自定义金额

## 使用指南

### 推荐方式：基于产品类型（规范化）

```typescript
import { useCommission } from '../composables/useCommission'

const { addSalesCommissionByProduct } = useCommission()

// 优点：
// 1. 价格预置，不会输入错误
// 2. 代码可读性强
// 3. 便于统计和管理
const records = addSalesCommissionByProduct(
  'TXN001',
  'C001',
  '张三',
  'offline',
  'L2标准管理',  // ← 产品类型，自动获取价格12800
  '2026-03-15',
  'DOC001',
  'COACH001'
)
```

### 灵活方式：基于自定义金额

```typescript
import { useCommission } from '../composables/useCommission'

const { addSalesCommission } = useCommission()

// 适用场景：
// 1. 折扣价格
// 2. 特殊协议价
// 3. 非标准产品
const records = addSalesCommission(
  'TXN001',
  'C001',
  '张三',
  'offline',
  10000,  // ← 自定义金额
  '2026-03-15',
  'DOC001',
  'COACH001'
)
```

## 角色说明

### 教练的双重收益

| 角色 | 收益类型 | 计算依据 | 金额范围 |
|-----|---------|---------|---------|
| **服务角色** | 管理费 | 用户数量 × 服务时长 | 100-400元/月/用户 |
| **销售角色** | 佣金提成 | 成交金额 × 提成比例 | 5%-7% |

**示例**：
```
教练张三某月的收益：
1. 管理费：
   - 服务5个新用户：5 × 400 = 2000元
   - 服务10个老用户：10 × 100 = 1000元
   小计：3000元

2. 佣金提成：
   - 线下成交L2标准管理：12800 × 5% = 640元
   - 线上成交线上高级版：7980 × 7% = 558.6元
   小计：1198.6元

该月总计：4198.6元
```
