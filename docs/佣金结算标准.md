# 佣金结算标准文档

## 更新日期
2026-03-16

## 重要说明

**佣金与管理费的区别**：
- **佣金**：与销售成交有关，包括线下成交提成、线上成交提成
- **管理费**：与服务用户的个数和服务日期有关，包括健康教练管理费、师父监管费、医生管理费

本文档仅说明佣金结算标准。管理费规则请参考《管理费结算标准文档》。

## 1. 销转提成结算

### 1.1 线下成交提成10%
- **结算标准**: 医生5%，销售5%，成交客户市场人员服务费200元
- **适用人员**: 医生、销售、**教练**（教练可作为销售人员获得提成）
- **分配方案**:
  - 医生提成：`成交金额 × 5%`
  - 销售/教练提成：`成交金额 × 5%`
  - 市场人员服务费：200元（固定）
- **总计**: 成交金额的10% + 200元服务费

### 1.2 线上成交提成10%
- **结算标准**: 销售/教练5%，医生3%，销售/教练2%
- **适用人员**: 医生、销售、**教练**（教练可作为销售人员获得提成）
- **分配方案**:
  - 销售/教练主要提成：`成交金额 × 5%`
  - 医生提成：`成交金额 × 3%`
  - 销售/教练辅助提成：`成交金额 × 2%`
- **总计**: 成交金额的10%

### 计算示例
```
线下成交5000元（教练张三 + 医生李四）：
- 医生提成：5000 × 5% = 250元（李四）
- 教练提成：5000 × 5% = 250元（张三）
- 市场人员服务费：200元
- 总支付：250 + 250 + 200 = 700元

线上成交5000元（教练张三 + 医生李四）：
- 教练主要提成：5000 × 5% = 250元（张三）
- 医生提成：5000 × 3% = 150元（李四）
- 教练辅助提成：5000 × 2% = 100元（张三）
- 总支付：250 + 150 + 100 = 500元
```

## 2. 结算注意事项

### 2.1 结算周期
- 佣金结算按月进行
- 每月7日前完成上个月佣金的统计和确认
- 每月10日完成支付

### 2.2 审批流程
- 销售人员提交成交记录
- 系统自动计算提成金额
- 主管审批确认
- 财务支付

### 2.3 特殊情况处理
- 如有退费情况，已发放的佣金需要追回
- 如有争议，请及时与相关负责人沟通

## 4. 技术实现

### 4.1 数据结构
佣金记录包含以下信息：
- 佣金类型（线上成交/线下成交/服务费）
- 收款人信息
- 成交客户信息
- 佣金金额
- 成交金额
- 结算周期
- 状态（待审批/已审批/已支付）

### 4.2 计算函数
```typescript
import { useCommission } from '../composables/useCommission'

const {
  // 推荐使用：基于产品类型（规范化）
  addSalesCommissionByProduct,  // 通过产品类型添加佣金
  getProductPriceConfig,        // 获取产品价格配置
  getProductPrice,              // 获取指定产品价格

  // 灵活使用：基于自定义金额
  addSalesCommission,           // 手动输入金额添加佣金
  getSalesCommissionConfig,     // 获取提成比例配置
  getCommissionStats
} = useCommission()

// ===== 方式1：基于产品类型（推荐） =====
const commissions = addSalesCommissionByProduct(
  'TXN001',           // 交易ID
  'C001',             // 客户ID
  '张三',             // 客户姓名
  'offline',          // 成交类型：'offline' | 'online'
  'L2标准管理',       // 产品类型（自动获取价格12800元）
  '2026-03-15',       // 成交日期
  'DOC001',           // 医生ID（可选）
  'COACH001'          // 教练/销售ID（可选，教练可作为销售）
)
// 自动生成：医生640元 + 销售640元 + 市场人员200元

// ===== 方式2：基于自定义金额（灵活） =====
const commissions2 = addSalesCommission(
  'TXN002',
  'C002',
  '李四',
  'online',
  15000,              // 自定义成交金额
  '2026-03-15',
  'DOC001',
  'COACH001'
)
// 自动生成：教练750元 + 医生450元 + 教练300元
```

## 6. 更新说明

1. ✅ **销转提成功能已移至** `useCommission()` composable
