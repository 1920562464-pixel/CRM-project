# AI 对话功能使用指南

## 🎉 功能介绍

AI 对话功能现在支持**真实的 API 调用**，不再只是模拟响应！

### ✨ 主要特性

- ✅ **多种 AI 服务支持**
  - OpenAI (GPT-3.5/GPT-4)
  - Anthropic Claude
  - 自定义 API（支持国内 AI 服务）

- ✅ **智能对话**
  - 上下文记忆
  - 专业健康管理助手
  - 支持 Markdown 格式

- ✅ **安全可靠**
  - API Key 本地存储
  - 模拟模式可选
  - 配置灵活切换

---

## 🚀 快速开始

### 方式一：使用模拟模式（无需配置）

默认情况下，AI 对话处于**模拟模式**，可以立即体验：

1. 访问 http://localhost:3000/
2. 点击"AI对话"或"智能助手"
3. 输入问题，如：
   - "王磊最近的血糖怎么样？"
   - "今日任务完成情况"
   - "查找糖尿病干预话术"

模拟模式会返回预设的回答，适合演示和测试。

---

### 方式二：配置真实 API

#### 1️⃣ 使用 OpenAI (GPT)

1. 点击左侧边栏的"配置 API"按钮
2. 切换"使用真实 API"开关
3. 选择 AI 服务商：**OpenAI (GPT-3.5/4)**
4. 输入 API Key（从 https://platform.openai.com/api-keys 获取）
5. 可选：自定义模型（默认 `gpt-3.5-turbo`）
6. 点击"保存配置"

**推荐模型**：
- `gpt-3.5-turbo` - 经济实惠，速度快
- `gpt-4` - 更强大，但价格更高

#### 2️⃣ 使用 Anthropic Claude

1. 点击"配置 API"
2. 切换"使用真实 API"开关
3. 选择 AI 服务商：**Anthropic (Claude)**
4. 输入 API Key（从 https://console.anthropic.com/ 获取）
5. 可选：自定义模型（默认 `claude-3-sonnet-20240229`）
6. 点击"保存配置"

#### 3️⃣ 使用自定义 API（国内服务）

如果你使用国内的 AI 服务（如通义千问、文心一言、讯飞星火等）：

1. 选择 AI 服务商：**自定义 API**
2. 输入 API Key
3. 输入 API Base URL（如 `https://dashscope.aliyuncs.com/compatible-mode/v1`）
4. 输入模型名称（如 `qwen-turbo`）

## 📝 API Key 获取指南

### OpenAI API Key

1. 访问 https://platform.openai.com/
2. 登录或注册账号
3. 进入 API Keys 页面
4. 点击"Create new secret key"
5. 复制生成的 Key（只显示一次！）

### Anthropic API Key

1. 访问 https://console.anthropic.com/
2. 登录或注册账号
3. 进入 API Keys 页面
4. 点击"Create Key"
5. 设置权限并复制 Key

---

## 🔒 安全说明

### 数据存储
- ✅ API Key 存储在**浏览器本地存储**（localStorage）
- ✅ 不会发送到任何第三方服务器
- ✅ 仅在你的设备上使用

### 安全建议
- 🔐 不要在公共设备上保存 API Key
- 🔐 建议在 API 服务商后台设置使用额度限制
- 🔐 定期更换 API Key
- 🔐 如果 API Key 泄露，立即在服务商后台删除并重新生成

AI 会记住对话历史，支持连续对话。你可以：

- 提出后续问题
- 要求详细说明
- 请求重新生成回答

### 快捷操作

- **Enter** - 发送消息

- **复制按钮** - 复制 AI 回答
- **重新生成** - 让 AI 重新回答
- **点赞/点踩** - 反馈回答质量（功能保留）

## 🆘 故障排除

### 问题 1：API 调用失败

**可能原因**：
- API Key 错误
- 网络连接问题
- API 服务不可用
- 模型名称错误

**解决方法**：
1. 检查 API Key 是否正确
2. 检查网络连接
3. 访问服务商状态页面
4. 切换回模拟模式测试

### 问题 2：响应速度慢

**可能原因**：
- 使用了高级模型（如 GPT-4）
- 网络延迟
- API 服务负载高

**解决方法**：
1. 切换到更快的模型（如 gpt-3.5-turbo）
2. 检查网络连接
3. 稍后重试

### 问题 3：回答质量不佳

**解决方法**：
1. 尝试切换到更强大的模型（如 GPT-4、Claude 3.5 Sonnet）
2. 提供更详细的问题背景
3. 分步骤提问
